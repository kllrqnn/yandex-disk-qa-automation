# yandex-disk-qa-automation
Тестовое задание на пример проекта автотестов для сервиса Яндекс.Диска 

---

## Стек технологий
* **Язык:** Python 3.10+
* **Фреймворк:** `pytest`
* **Библиотеки:** `requests` (API), `allure-pytest` (отчеты), `Ruff/black` (качество кода)
* **Логирование:** Logging 

---

## Структура проекта
**clients/:** - слой взаимодействия с API.  
- `base_client.py` - базовый клиент с обшей логикой. 
- `disk_client.py` - клиент для работы с ресурсами диска. 
- `trash_client.py` - клиент для управления с корзиной. 
**files/:** - статистические тестовые данные. 
- `test_sample.txt` - тестовый файл для проверки функционала загрузки в диск. 
**fixtures/:** - глобальные фикстуры pytest. 
- `settings.py` - фикстура для инициализации обьекта настроек из конфигурации. 
**tests/:** - тестовые сценарии. 
- `conftest.py` - главный файл конфигурации тестов. 
- `test_disk_api.py` - набор функциональных тестов. 
**tools/:** - вспомогательные инструменты и утилиты. 
- `assertions/disk_assertions.py` - кастомные проверки для валидации бизнес-логики ответов. 
- `fakers.py` - генератор случайных данных для уникальных запусков. 
- `logger.py` - настройка конфигурации логирования. 
- `routes.py` - перечисление всех эндпоинтов.  
**config.py** - описание модели настроек, загрузка переменных окружения из .env. 
**pyproject.toml** - настройки инструментов линтинга и форматирования. 
**pytest.ini** - конфигурация запусков pytest. 
**requirements.txt** - список зависимостей проекта.

## Запуск проекта

### Настройка переменных окружения
Создайте в корне проекта файл `.env` и добавьте в него ваш OAuth-токен:
`YANDEX_DISK_TOKEN=ваш_токен`

### Подготовка окружения
Клонируйте репозиторий и создайте виртуальное окружение:
```bash
git clone <https://github.com/kllrqnn/yandex-disk-qa-automation.git>
python3 -m venv venv
source venv/bin/activate #для Linux/Mac 
source venv\Scripts\activate # для Windows
```
### Установка зависимостей
```bash
pip install -r requirements.txt
```
### Запуск тестов
```bash
pytest tests/ # для обычного запуска
pytest --alluredir=allure-results # для запуска с генерацией данных для Allure-отчета
```
### Запуск Allure
```bash
allure serve allure-results
```
